## Неизменяемые инварианты (каждая строка = проверяемое утверждение)

1. `Rules_POWER.md` содержит полный список неизменяемых инвариантов проекта.
2. `README.md` содержит спецификацию и обоснования, и не должен противоречить `Rules_POWER.md`.
3. При расхождении кода и `Rules_POWER.md` считать код неверным и приводить к `Rules_POWER.md`.
4. При расхождении `README.md` и `Rules_POWER.md` привести документы к согласованию, сохранив `Rules_POWER.md`.
5. Любые изменения конфигурации периферии выполняются только через `.ioc` (CubeMX).
6. Пользовательский код в автогенерируемых файлах размещается только в `/* USER CODE BEGIN */ ... /* USER CODE END */`.
7. Логика power sequencing реализуется только через state machine и счётчик времени `systick_ms` без `HAL_Delay()`.
8. ISR/IRQ содержат только минимальную работу и не содержат тяжёлой логики state machine.
9. В проекте запрещено использование `float` в прикладной логике (Cortex‑M0).
10. Все тайминги рассчитываются исходя из `SYSCLK = 32 МГц`.
11. UART0 имеет параметры `115200 8N1` без flow control.
12. Фрейм UART0 имеет вид `[STX][CMD][LEN][DATA][CRC][ETX]` с `STX=0x02` и `ETX=0x03`.
13. Поле `LEN` в UART0 равно длине `DATA` в байтах.
14. CRC в UART0 равен `CRC-8/ATM` по байтам `[CMD][LEN][DATA...]` (poly 0x07, init 0x00, refin=false, refout=false, xorout 0x00).
15. Endianness всех многобайтных полей протокола UART0 — Little Endian.
16. Команда `PING (0x01)` всегда отвечает `status=0xAA`.
17. Ответ `GET_STATUS (0x04)` всегда имеет `LEN=26`.
18. Порядок полей `GET_STATUS` неизменяемый: `v24,v12,v5,v3v3,i_lcd,i_bl,i_scaler,i_audio_l,i_audio_r,temp0,temp1,state,fault_flags,inputs`.
19. Поле `state` в `GET_STATUS` является битовой маской доменов в формате `POWER_CTRL` (bits 0..6).
20. Поле `fault_flags` в `GET_STATUS` является защёлкнутой (latched) маской причин fault.
21. Команда `RESET_FAULT (0x05)` очищает `fault_flags`, но не включает домены автоматически.
22. Fault в системе является latched и не сбрасывается автоматически по таймеру или по нормализации измерений.
23. `BACKLIGHT` запрещено включать, если `SCALER` выключен или `LCD` выключен.
24. При запросе `SCALER=OFF` или `LCD=OFF` при включённом `BACKLIGHT` выполняется корректный shutdown sequencing без прямого выключения.
25. Safe state на старте обязателен до обработки любых команд UART и до любых секвенсов.
26. В safe state все домены питания (active HIGH) установлены в OFF (GPIO LOW).
27. В safe state линии `PWRBTN#` и `RSTBTN#` (open-drain) находятся в release (не тянут в LOW).
28. В safe state усилитель удерживается в `SDZ=LOW` (shutdown) и `MUTE=HIGH` (mute).
29. `BACKLIGHT` не включается автоматически при старте, если явно не запрошено по UART.
30. Работа АЦП реализована как ADC scan + DMA circular.
31. DMA буфер АЦП имеет длину ровно 14 элементов (`ADC_CHANNEL_COUNT = 14`).
32. Порядок ADC ranks 1..14 соответствует DMA index 0..13 и не изменяется.
33. DMA index 0..13 соответствует: `PA0,PA1,PA4,PA5,PA6,PA7,PB0,PB1,PC0,PC1,PC2,PC3,PC4,PC5`.
34. Опорное напряжение АЦП фиксировано как `VDDA = 2.5 В` и `ADC_VREF_MV = 2500`.
35. Пересчёт `raw → mV` при 12-bit Right alignment выполняется формулой `mv = raw * 2500 / 4096`.
36. При 12-bit Left alignment перед пересчётом выполняется `raw >>= 4`.
37. Температурные каналы `Temp0/Temp1` считаются резервом и могут возвращаться как невалидные значения.
38. При отсутствии NTC `temp0` и `temp1` в телеметрии устанавливаются в `-32768`.
39. Фильтрация аварий использует окно усреднения 8 измерений.
40. Подтверждение аварии выполняется по правилу “5 подряд измерений вне порога”.
41. Любое измерение в норме сбрасывает consecutive-счётчик подтверждения аварии.
42. При любом fault сначала выполняется перевод в safe state, затем фиксируются `fault_flags`.
43. При `PGOOD=LOW` секвенсы дисплея запрещены.
44. Падение `PGOOD` во время секвенса приводит к аварийному выключению дисплея и фиксации fault.
45. Команда `BOOTLOADER_ENTER (0x08)` перед reset переводит систему в safe state.
46. `BOOTLOADER_ENTER (0x08)` перед reset отправляет ACK и дожидается завершения передачи UART.
47. Вход в ROM bootloader выполняется jump на адрес `0x1FFF0000` (STM32F030) при обнаружении SRAM magic.
48. При включённом `IWDG` refresh выполняется ровно в одном месте main loop один раз за итерацию.
49. Refresh `IWDG` выполняется после обработки UART, ADC, state machine и fault-логики.
50. Refresh `IWDG` запрещён из ISR, HAL callback и низкоуровневых функций периферии.
